# Day 5: Collections and Generics

## Collections Framework
The Collection in Java is a framework that provides an architecture to store and manipulate a group of objects. It includes interfaces and their varying implementations.
### Key Interfaces:
- **List**: An ordered collection. Accepts duplicate values. Implementations: `ArrayList`, `LinkedList`.
- **Set**: A collection that cannot contain duplicate elements. Implementations: `HashSet`, `TreeSet`.
- **Map**: An object that maps keys to values. Keys cannot be duplicated. Implementations: `HashMap`, `TreeMap`.

## Generics
Java Generic methods and generic classes enable programmers to specify, with a single method declaration, a set of related methods, or with a single class declaration, a set of related types.
- Provides Compile-Time Type Safety.
- Eliminates the need for Type Casting.

## Iterators
The `Iterator` interface provides the facility of iterating the elements in a forward direction only. `ListIterator` for bidirectional.
